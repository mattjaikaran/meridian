  <name>Task 1: Build open_project, retry_on_busy, DatabaseBusyError, and backup_database in scripts/db.py</name>
  <files>scripts/db.py</files>
  <behavior>
    - open_project(path) yields a configured sqlite3.Connection
    - open_project sets WAL, foreign_keys, busy_timeout=5000 on every connection
    - open_project auto-commits on clean exit (no exception)
    - open_project rolls back on exception, then re-raises
    - open_project closes connection in finally block
    - open_project(":memory:") works for test fixtures (yields in-memory conn, skips close-reopen issues)
    - connect() becomes _connect() (internal only, prefixed with underscore)
    - retry_on_busy(max_retries=3, base_delay=0.5) retries on "database is locked" OperationalError
    - retry_on_busy raises DatabaseBusyError(retries, total_wait) after exhaustion
    - retry_on_busy does NOT retry other OperationalErrors (e.g., bad SQL)
    - retry_on_busy applies jitter of +/-25% on backoff delays
    - DatabaseBusyError stores retries count and total_wait time
    - backup_database(source_path) creates hot snapshot in source_path.parent/backups/
    - backup_database uses connection.backup() API
    - backup_database names files state-{ISO-timestamp-with-microseconds}.db
    - _prune_backups removes oldest files when count exceeds max_backups (default 100)
    - init_schema accepts optional db_path parameter for auto-backup before migration
  </behavior>
  <action>
Refactor scripts/db.py:

1. Add imports: `contextlib`, `functools`, `random`, `time`, `datetime` (UTC, datetime).

2. Create `DatabaseBusyError(Exception)` class with `retries: int` and `total_wait: float` attributes. Message format: `"Database busy after {retries} retries ({total_wait:.1f}s total wait)"`.

3. Create `retry_on_busy(max_retries=3, base_delay=0.5)` decorator per the pattern in 01-RESEARCH.md Pattern 2. Catch `sqlite3.OperationalError`, check `"database is locked" in str(e)`. Backoff: `base_delay * (2 ** attempt)` with `random.uniform(-0.25, 0.25) * delay` jitter. On exhaustion raise `DatabaseBusyError(...) from e`.

4. Rename `connect()` to `_connect()`. Add `conn.execute("PRAGMA busy_timeout=5000")` inside it. Keep all existing pragmas (WAL, foreign_keys).

5. Create `open_project(path=None)` as `@contextlib.contextmanager`. Pattern:
   - If path is `:memory:`, create in-memory connection directly (sqlite3.connect, set row_factory + pragmas, init_schema, yield, close).
   - Otherwise call `_connect(path)`, call `init_schema(conn, db_path=resolved_path)`, yield conn, commit on success, rollback on exception, close in finally.

6. Create `backup_database(source_path, backup_dir=None, max_backups=100)` per RESEARCH Pattern 3. Use microsecond-resolution timestamps (`%Y%m%dT%H%M%S_%fZ`). Return backup path.

7. Create `_prune_backups(backup_dir, max_backups)` to remove oldest beyond limit.

8. Update `init_schema(conn, db_path=None)` to accept optional db_path. Before running migrations (the `if current_version < 2:` block), call `backup_database(Path(db_path))` if db_path is provided and not `:memory:`.

9. Update `init()` function to use `open_project()` instead of manual connect/try/finally/close.

10. Update `__main__` block: the `version` command should use `open_project()` instead of manual connect/close.

11. Keep `get_db_path()`, `get_schema_version()`, `init_schema()` as public exports. Remove `connect` from public API (now `_connect`).

IMPORTANT: Do NOT remove per-function commits in state.py -- double-commit is harmless in SQLite per RESEARCH anti-patterns section.
  </action>
