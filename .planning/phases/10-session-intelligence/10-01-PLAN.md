<objective>
Build structured session handoff for richer context restoration.

When a user pauses work mid-session, capture machine-readable context:
1. `/meridian:pause` (or checkpoint with --handoff) creates `.meridian/HANDOFF.json`
2. JSON contains:
   - active_phase, active_plan (IDs + names)
   - blockers (from DB blocked status or user input)
   - decisions_made (recent decision records from DB)
   - files_modified (from git diff since last checkpoint)
   - next_action (from compute_next_action)
   - user_notes (optional freeform from user)
   - created_at timestamp
3. `/meridian:resume` checks for HANDOFF.json first
4. If found: merge handoff context into resume prompt (richer than DB-only resume)
5. After resume: delete HANDOFF.json (consumed)

This enriches the existing deterministic resume with session-specific context that
the DB alone can't capture (like "I was debugging X when I stopped").
</objective>
