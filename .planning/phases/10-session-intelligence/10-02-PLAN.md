<objective>
Build a persistent debug knowledge base that survives sessions.

After /meridian:debug resolves an issue, append structured findings:
```markdown
## [DBG-003] 2026-03-20 — VALIDATION.md filename mismatch

**Symptom:** Nyquist engine couldn't find any VALIDATION.md files
**Root Cause:** Hardcoded `VALIDATION.md` but real files use `NN-VALIDATION.md` prefix
**Fix:** Added `_find_validation_md()` glob helper matching `*VALIDATION.md`
**Files:** scripts/nyquist.py, tests/test_nyquist.py
```

When starting a new debug session, search the KB first — if a similar symptom
was seen before, surface the prior resolution as context.

Storage: simple markdown in `.meridian/debug-kb.md` (not in DB — too unstructured).
Search: keyword matching against symptom + root cause fields.
Dedup: hash of root_cause field to avoid duplicate entries.
</objective>
