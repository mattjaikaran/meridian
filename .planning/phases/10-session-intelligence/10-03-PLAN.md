<objective>
Add traceable decision IDs that link discussions to plans.

Currently decisions are free-text records. This adds:
1. Auto-generated decision_id (DEC-001, DEC-002, ...) on create_decision()
2. A junction table `plan_decision` linking plan_id ↔ decision_id
3. `link_decision_to_plan(conn, decision_id, plan_id)` function
4. `get_decisions_for_plan(conn, plan_id)` → list of decisions
5. When generating plan context for subagents, include linked decisions

Schema migration (v3→v4):
- ALTER TABLE decision ADD COLUMN decision_id TEXT UNIQUE
- CREATE TABLE plan_decision (plan_id TEXT, decision_id TEXT, FOREIGN KEY...)
- Backfill existing decisions with DEC-001, DEC-002, etc.
</objective>
