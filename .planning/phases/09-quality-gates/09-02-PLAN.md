<objective>
Add requirements coverage gate to scripts/gates.py.

Before executing a phase's plans, verify that every requirement assigned to the phase
has at least one plan that claims to address it (via the plan's `requirements` field).

Logic:
1. Get phase requirements from ROADMAP.md or DB
2. Get all plans for the phase with their requirement lists
3. Compute coverage: which reqs have plans, which don't
4. If uncovered reqs exist: warn (or block if strict mode)
5. Return coverage report dict
</objective>
