<objective>
Build a cross-phase regression gate that runs before phase execution.

When /meridian:execute starts a new phase, before any plans run:
1. Collect full_suite_command from all completed phases' VALIDATION.md files
2. Run each command sequentially
3. If all pass: proceed with execution
4. If any fail: block execution, show which phase regressed and the failing output
5. Log gate result as a state_event

Integration point: Hook into `transition_phase(conn, phase_id, 'executing')` in state.py — run gate check before allowing the transition.
</objective>
