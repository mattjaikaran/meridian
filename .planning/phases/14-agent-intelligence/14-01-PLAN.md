<objective>
Build MCP tool awareness so subagents can discover and use MCP server tools.

When generating subagent prompts for plan execution:
1. Check for available MCP servers (via environment or config)
2. List available tools from each server
3. Score each tool's relevance to the current plan (keyword match on description + plan)
4. Include relevant tool descriptions in the subagent prompt
5. Cache results for the session

This lets subagents use tools like web search, database access, or API calls
when they're available, without requiring explicit configuration per plan.

Implementation notes:
- MCP tool listing is done via the protocol's list_tools endpoint
- For now, simulate via a config file `.meridian/mcp-tools.json` listing available tools
- Real MCP integration can be wired in later
</objective>
