  <name>Task 1: Build roadmap_sync.py with TDD</name>
  <files>scripts/roadmap_sync.py, tests/test_roadmap_sync.py</files>
  <behavior>
    - sync_roadmap_plan_checkbox(text, plan_slug, is_complete) -> str:
      - Input: full ROADMAP.md text, plan slug like "05-01-PLAN.md", bool
      - When is_complete=True: changes `- [ ] 05-01-PLAN.md` to `- [x] 05-01-PLAN.md`
      - When is_complete=False: changes `- [x] 05-01-PLAN.md` to `- [ ] 05-01-PLAN.md`
      - Unrelated lines unchanged
      - Returns original text if plan slug not found (no crash)

    - sync_roadmap_phase_checkbox(text, phase_number, is_complete) -> str:
      - Input: full ROADMAP.md text, phase number (int), bool
      - Targets: `- [ ] **Phase N:` lines in milestone sections
      - When is_complete=True: `- [ ]` -> `- [x]`
      - When is_complete=False: `- [x]` -> `- [ ]`
      - Returns original text if phase line not found

    - sync_roadmap_progress_table(text, phase_number, status, completed_date) -> str:
      - Input: full ROADMAP.md text, phase number (int), status string, optional date string
      - Targets progress table row: `| N. Phase Name | ... | Status | Date |`
      - Updates Status column and Completed column
      - completed_date=None -> "-" in date column
      - Returns original text if row not found

    - sync_requirements_status(text, req_id, new_status) -> str:
      - Input: full REQUIREMENTS.md text, req ID like "ROAD-01", status string
      - Targets traceability table row: `| ROAD-01 | Phase 7 | Pending |`
      - Updates status column to new_status (e.g., "Complete")
      - Returns original text if req_id not found

    - Edge cases:
      - Multiple plan lines exist — only the matching slug is updated
      - Already-correct checkbox state — text returned unchanged (idempotent)
      - Completely empty text input — returns empty string, no crash
  </behavior>
