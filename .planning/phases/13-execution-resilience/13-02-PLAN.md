<objective>
Build node repair operators for automatic recovery when task execution fails.

When a plan fails (test command returns non-zero), instead of immediately failing the phase:
1. Check repair budget (default: 2 attempts remaining)
2. Analyze the failure:
   - Test failure with clear error → RETRY with error context
   - Task too large / multiple unrelated failures → DECOMPOSE into sub-plans
   - Task is non-critical / already partially done → PRUNE and continue
3. Execute the chosen strategy
4. Decrement repair budget
5. If budget exhausted → fail the plan normally

Strategy selection heuristic:
- First failure → always RETRY
- Second failure, same error → DECOMPOSE if possible, else PRUNE
- Non-test failure (syntax error, import error) → RETRY with fix hint
</objective>
