<objective>
Build /meridian:pr-branch to create clean PR branches without planning artifacts.

Problem: Meridian commits include .planning/ and .meridian/ changes that reviewers don't need to see. This command creates a clean branch for PR review.

Approach:
1. Identify commits on current branch since divergence from main
2. For each commit, check if it touches files outside .planning/ and .meridian/
3. Cherry-pick only "code" commits onto a new `pr/<slug>` branch
4. For mixed commits (code + planning), create a filtered version using `git diff` + `git apply`
5. Push the clean branch

Edge cases:
- All commits are planning-only → warn, no branch created
- Single commit with mixed changes → split into code-only patch
- Merge conflicts during cherry-pick → abort and explain
</objective>
