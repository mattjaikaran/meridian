<objective>
Build a centralized security module consolidating all input validation.

Functions:
1. `validate_path(path, project_root)` — Resolve path, reject if it escapes project_root via `../` or symlinks. Returns resolved Path or raises MeridianError.
2. `safe_json_loads(text)` — json.loads with size limit and error handling. Returns parsed dict or None.
3. `validate_field_name(name)` — Regex check for SQL-safe identifiers `[a-zA-Z_][a-zA-Z0-9_]*`. Used by safe_update().
4. `sanitize_shell_arg(arg)` — Reject or quote args containing `;`, `|`, `&`, `$`, backticks, etc. For subprocess commands.

Integration:
- Replace inline path checks in state.py/db.py with validate_path()
- Replace bare json.loads() calls with safe_json_loads()
- Wire validate_field_name() into safe_update() (may already be similar)
- Use sanitize_shell_arg() in nyquist.py subprocess calls
</objective>
