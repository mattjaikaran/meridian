<feature>
  <name>Command Generator Script</name>
  <files>scripts/generate_commands.py, tests/test_generate_commands.py</files>
  <behavior>
    - discover_skills(repo_root) returns list of 13 dicts with name, description, argument_hint keys
    - extract_metadata(skill_md_path) parses title line for description, Arguments section for hints
    - extract_metadata handles SKILL.md with no Arguments section (returns empty argument_hint)
    - generate_wrapper(skill_metadata) produces string with marker, frontmatter, description, @ reference
    - Generated wrapper first line is exactly `<!-- meridian:generated -->`
    - Generated wrapper @ reference uses absolute path: `/Users/mattjaikaran/.claude/skills/meridian/skills/<name>/SKILL.md`
    - is_generated(filepath) returns True for generated files, False for custom files
    - write_commands(skills, commands_dir) writes .md files, skips custom files, returns created/skipped counts
    - cleanup_orphans(commands_dir, valid_names) removes generated files not in valid_names set
    - update_root_skill(repo_root, skills) regenerates SKILL.md without Commands section, keeps Architecture/Scripts/References
    - verify_symlink(expected_target) returns True if symlink exists and points to correct target
    - fix_symlink(symlink_path, target) creates or recreates symlink
    - uninstall(commands_dir) removes only generated files, prints summary
    - main() with argparse: default=install, --uninstall, --fix-symlink, --all (with --uninstall)
    - main() prints summary: "Created: init.md, plan.md, ..." and "Skipped: N custom files" and "Removed: N orphans"
  </behavior>
  <implementation>
    Build scripts/generate_commands.py as a single-file stdlib-only Python script.

    Key implementation details:
    - Use pathlib throughout for path operations
    - Use argparse for CLI: default action is install, --uninstall removes generated files, --fix-symlink repairs symlink, --all with --uninstall also removes symlink
    - Hardcode REPO_ROOT = Path("/Users/mattjaikaran/dev/meridian") (personal tool, per user decision)
    - Hardcode COMMANDS_DIR = Path.home() / ".claude" / "commands" / "meridian"
    - Hardcode SYMLINK_PATH = Path.home() / ".claude" / "skills" / "meridian"
    - GENERATED_MARKER = "<!-- meridian:generated -->"
    - Parse SKILL.md title: regex r'[—–-]\s*(.+)$' on first line starting with "# "
    - Parse Arguments section: lines between "## Arguments" and next "## " heading, extract backtick content for hint
    - Root SKILL.md template: keep current Architecture, Scripts, References sections as a template string in the script. Add an informational "Available Commands" list (without /meridian: invocation syntax to avoid routing conflict). Remove the current Commands section entirely.
    - All tests use tmp_path fixtures -- never touch real ~/.claude/ or real skills/
    - Tests create minimal fake SKILL.md files in tmp_path to test parsing
  </implementation>
</feature>
