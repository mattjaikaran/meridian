<objective>
Build /meridian:fast for trivial inline tasks that skip the full planning pipeline.

This is for one-liner changes, typo fixes, small config tweaks — anything that doesn't warrant a phase/plan/subagent. The command should:
1. Accept freeform text describing the task
2. Run a quick complexity heuristic (file count, keywords like "refactor", "architect")
3. If trivial: execute inline, commit with `[meridian:fast]` prefix
4. If complex: warn user and suggest `/meridian:quick` or `/meridian:plan`
5. Log a lightweight event to the DB (fast_task table or state_event)
</objective>
