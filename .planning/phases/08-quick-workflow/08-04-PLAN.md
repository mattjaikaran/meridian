<objective>
Build /meridian:next as an auto-advance command that detects where you are and moves forward.

This wraps the existing `compute_next_action()` from state.py with a user-facing layer:
1. Query current state via compute_next_action()
2. Map the action to a specific /meridian:* command with args
3. Display what it's about to do and why
4. Execute the mapped command (or ask for confirmation on destructive actions)

State → Action mapping:
- no_project → "Run /meridian:init to get started"
- create_phases → "Run /meridian:plan to create phases"
- plan_phase → "Run /meridian:plan --phase {id}"
- execute / execute_plan → "Run /meridian:execute"
- verify_phase → "Run /meridian:execute (verification step)"
- complete_milestone → "Run milestone completion"
- idle → "All caught up. Use /meridian:note to capture ideas."
</objective>
