# Technology Stack

**Analysis Date:** 2026-03-10

## Languages

**Primary:**
- Python 3.11+ (required minimum), running on Python 3.14.3 locally - All application code

**Secondary:**
- Bash - External PM tool integration (`~/zeroclaw/skills/kanban/pm.sh`)
- Markdown - Skill definitions (`skills/*/SKILL.md`), prompt templates (`prompts/*.md`), reference docs (`references/*.md`)

## Runtime

**Environment:**
- CPython 3.11+ (specified in `pyproject.toml` `requires-python = ">=3.11"`)
- Locally running Python 3.14.3

**Package Manager:**
- uv 0.7.17
- Lockfile: `uv.lock` present (revision 2)

## Frameworks

**Core:**
- None - stdlib-only Python project. No web framework, no ORM. Uses `sqlite3`, `json`, `pathlib`, `datetime`, `textwrap`, `subprocess`, `urllib` from stdlib.

**Testing:**
- pytest - Test runner (imported in `tests/test_state.py`)

**Build/Dev:**
- ruff - Linting and formatting (`pyproject.toml` `[tool.ruff]`)

## Key Dependencies

**Critical:**
- Zero external dependencies. The `uv.lock` contains only the `meridian` package itself. All functionality is built on Python stdlib.

**Infrastructure:**
- SQLite3 (stdlib) - Primary data store via `scripts/db.py`
- urllib.request (stdlib) - HTTP client for Nero RPC calls in `scripts/dispatch.py` and `scripts/sync.py`
- subprocess (stdlib) - Git operations in `scripts/state.py`, `scripts/resume.py`; Axis PM script execution in `scripts/axis_sync.py`

## Configuration

**Environment:**
- No `.env` files detected. No environment variables used directly.
- Configuration is stored in the SQLite database `project` table: `nero_endpoint`, `axis_project_id`, `tech_stack`, `repo_path`, `repo_url`
- Configuration is set during `/meridian:init` and persisted to `.meridian/state.db`

**Build:**
- `pyproject.toml` - Project metadata, ruff config
- ruff settings: `line-length = 100`, `target-version = "py311"`, lint rules: `["E", "F", "I", "N", "W", "UP"]`

## Platform Requirements

**Development:**
- Python 3.11+
- uv package manager
- git (required for checkpoint git state tracking, resume prompt generation)
- SQLite3 (bundled with Python)

**Production:**
- This is a CLI/skill tool consumed by Claude Code, not a deployed service
- Runs locally on developer machines alongside Claude Code
- Each project gets its own `.meridian/` directory with `state.db`

## Data Storage

**SQLite Database:**
- Location: `<project_dir>/.meridian/state.db`
- Schema version: 2 (with migration support in `scripts/db.py`)
- WAL journal mode enabled
- Foreign keys enforced
- Tables: `schema_version`, `project`, `milestone`, `phase`, `plan`, `checkpoint`, `decision`, `nero_dispatch`, `quick_task`

**JSON Export:**
- Location: `<project_dir>/.meridian/meridian-state.json`
- Generated by `scripts/export.py` for git tracking and Nero consumption
