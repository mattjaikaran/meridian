# Meridian Resume Template

This template shows the structure of a resume prompt. The actual content is generated by `scripts/resume.py` from SQLite queries.

```
# Meridian Resume — {project_name}

## Position
- Milestone: {milestone_name} ({milestone_status})
- Phase {N}: {phase_name} ({phase_status})
- Current plan: {plan_name} ({plan_status})

## Phase Overview
→ Phase 1: Foundation [complete] (3/3 plans)
  Phase 2: Features [executing] (1/4 plans)
  Phase 3: Polish [planned] (0/0 plans)

## Current Phase: {phase_name}
{phase_description}

## Acceptance Criteria
- [ ] criterion 1
- [x] criterion 2

## Completed in This Phase
- Plan A (commit: abc1234)
- Plan B (commit: def5678)

## Remaining Plans
- Wave 2: Plan C
  > Description of what Plan C does...
- Wave 2: Plan D **[failed]** — Error: tests broke
  > Description of what Plan D does...

## Key Decisions (Recent)
- [architecture] Chose SQLite over filesystem for state
- [approach] TDD required for all database operations

## Blockers
- (none, or list of current blockers)

## Git State
Branch: feature/add-auth, SHA: abc1234
Recent commits:
  abc1234 [meridian] Add user model and migrations
  def5678 [meridian] Setup test infrastructure

## Next Action
→ Execute plan: 'Plan C' (wave 2)
  Plan: Add API routes
  Wave: 2
```

## Design Principles

1. **Deterministic**: Same database state = same prompt. No randomness, no LLM prose.
2. **Complete**: Contains everything needed to resume work. No external references needed.
3. **Structured**: Easy to parse. Each section maps to a specific DB query.
4. **Actionable**: Always ends with a computed next action. The resuming agent knows exactly what to do.
