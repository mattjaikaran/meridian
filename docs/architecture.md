# Architecture Guide

How Meridian works under the hood.

## Design Principles

1. **Deterministic state** — All state lives in SQLite. Same DB = same behavior.
2. **Fresh context per task** — Each plan gets a 200k-token subagent. No context rot.
3. **Stdlib only** — Zero external dependencies. Works everywhere Python does.
4. **Quality by default** — Gates, detection, and review are automatic, not opt-in.
5. **Graceful degradation** — Every integration (Nero, MCP, Axis) fails silently.

## Command Routing

```mermaid
graph LR
    A[User types /meridian:plan] --> B[~/.claude/commands/meridian/plan.md]
    B -->|@ reference| C[skills/plan/SKILL.md]
    C -->|procedure calls| D[scripts/state.py<br>scripts/db.py]
```

1. Claude Code finds `plan.md` in `~/.claude/commands/meridian/`
2. The `.md` wrapper references the SKILL.md via `@` directive
3. SKILL.md contains the procedure (Python calls, bash commands, steps)
4. Procedure calls `scripts/*.py` modules for all logic

Command wrappers are auto-generated by `scripts/generate_commands.py` from `skills/*/SKILL.md` metadata.

## State Machine

### Phase Lifecycle

```mermaid
stateDiagram-v2
    [*] --> planned
    planned --> context_gathered: gather context
    context_gathered --> planned_out: create plans
    planned_out --> executing: start execution
    executing --> verifying: all plans done
    verifying --> reviewing: verification pass
    reviewing --> complete: review pass

    planned --> blocked: blocker found
    context_gathered --> blocked: blocker found
    planned_out --> blocked: blocker found
    executing --> blocked: blocker found
    blocked --> planned: unblock
    blocked --> context_gathered: unblock
    blocked --> planned_out: unblock
    blocked --> executing: unblock
```

### Plan Lifecycle

```mermaid
stateDiagram-v2
    [*] --> pending
    pending --> executing: start
    executing --> complete: tests pass
    executing --> failed: tests fail
    failed --> pending: retry (node repair)
    pending --> skipped: prune
    executing --> paused: user pause
    paused --> executing: resume
```

## Execution Flow

```mermaid
sequenceDiagram
    participant U as User
    participant M as Meridian
    participant DB as SQLite
    participant G as Gates
    participant A as Subagent

    U->>M: /meridian:execute
    M->>DB: compute_next_action()
    DB-->>M: execute phase 2, wave 1

    M->>G: run_regression_gate(phase 2)
    G->>G: Run phases 1 tests
    G-->>M: PASS

    M->>G: check_requirements_coverage()
    G-->>M: All reqs covered

    loop For each plan in wave 1
        M->>DB: transition_plan(executing)
        M->>A: Spawn with 200k tokens + plan context
        A->>A: TDD: red → green → refactor
        A->>A: git commit
        A-->>M: Complete (commit SHA)
        M->>DB: transition_plan(complete)
        M->>G: detect_stubs(modified files)
    end

    M->>DB: check_auto_advance()
    DB-->>M: All plans done → verifying
    M->>M: Two-stage review
    M->>DB: transition_phase(complete)
    M-->>U: Phase 2 complete. Next: phase 3
```

## Quality Gate Pipeline

```mermaid
graph TB
    Start[Phase Execution Start] --> RG{Regression Gate}
    RG -->|PASS| RC{Requirements Coverage}
    RG -->|FAIL| Block[Block Execution<br>Show failing phase]
    RC -->|All covered| Exec[Execute Plans]
    RC -->|Gaps found| Warn[Warn + Continue]
    Warn --> Exec
    Exec --> SD{Stub Detection}
    SD -->|Clean| Review[Two-Stage Review]
    SD -->|Stubs found| WarnStubs[Warn + Review]
    WarnStubs --> Review
    Review --> Complete[Phase Complete]
```

## Node Repair Flow

```mermaid
graph TB
    Fail[Plan Failed] --> Budget{Repair Budget > 0?}
    Budget -->|No| FailPlan[Mark Failed]
    Budget -->|Yes| Analyze{Analyze Failure}
    Analyze -->|First failure| Retry[RETRY<br>Re-execute with error context]
    Analyze -->|Same error twice| Decompose[DECOMPOSE<br>Split into sub-plans]
    Analyze -->|Non-critical| Prune[PRUNE<br>Skip and continue]
    Retry --> Result{Success?}
    Decompose --> Result
    Result -->|Yes| Complete[Plan Complete]
    Result -->|No| Budget
    Prune --> Continue[Continue remaining plans]
```

## File Organization

```
.meridian/                    # Per-project state (gitignored)
├── state.db                  # SQLite database (source of truth)
├── backups/                  # Auto-backups before migrations
├── notes.md                  # Captured notes
├── backlog.md                # Seeds with triggers
├── debug-kb.md               # Debug knowledge base
├── DISCUSSION-LOG.md         # Decision audit trail
├── HANDOFF.json              # Session handoff (temporary)
├── USER-PROFILE.md           # Developer profile
└── mcp-tools.json            # MCP tool config (optional)
```
